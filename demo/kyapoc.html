👂 क्या लिखा है — "What does it say?"

Tap the ear icon next to any heading. The system confirms with a ting, classifies the content, plays a distinct tone, then speaks the explanation aloud.

1
Ting (880Hz, 80ms) — auditory confirmation. "I heard you."
2
Classification — Gemini infers informational / caution / critical. Distinct tone per class.
3
Explanation — AI generates a plain-language summary. Sarvam Bulbul speaks it aloud in the active language.
4
ARIA live region updates at each step — screen readers announce in parallel.

📷 Scan Button — Camera + Face Auth

The central FAB breaks the nav grid — spatial signal for primary action. Two paths through one button:

A
QR / Document Scan — Camera opens immediately with viewfinder and voice guidance.
B
Face Authentication — DPDP Act consent capture first, then camera with face oval guide and timed voice prompts.

🔊 Voice Guidance Engine

Shared service across both patterns. Queued announcements — never overlapping. Minimum 1.5s gap. aria-live="polite" throughout. Tones via Web Audio API (no files needed). TTS via Sarvam Bulbul for natural Indic speech.

The Proposal

A Design System as Digital Public Infrastructure

India built identity infrastructure the world studies. India built payment infrastructure the world adopts. India has never built experience infrastructure — the shared layer that makes interacting with the state legible, consistent, and dignified across every touchpoint.

🎯

Token-Driven

W3C Design Tokens format. Primitive → semantic → component layers. Style Dictionary compiles to CSS, iOS Swift, Android Compose from one source of truth.

🌐

14 Languages, 13 Scripts

Per-script expansion factors, line-height multipliers, RTL support for Urdu. The bhāṣā problem solved as a first-class token primitive.

♿

Population-Scale Access

56px touch targets for calloused hands. 2G-functional. ₹6,000 device baseline. WCAG 2.1 AA minimum, AAA target. AI-driven reading assistant for neo-literate users.

📦

Containerised Commons

Docker container + npm package. Departments install, theme via brand tokens, ship. Fork-and-extend model. Git-based governance. Observable adoption.

🔐

Three-Tier Governance

Canonical: UIDAI controls primitives. Extended: Departments create semantic tokens. Consumer: Self-service theming. Accessibility tokens never overridable.

🤖

Generative-Ready

The canonical token layer becomes the constraint grammar for AI-generated UIs. Guardrails for generative public services. Future-proof by architecture, not by prediction.

Architecture

How It Deploys — Design System as India Commons

From UIDAI's master design system to every government digital service. The pattern follows India Stack's proven rail model: build once, offer as infrastructure, let adoption prove the case.

How this demo maps to the architecture: The Kya Likha Hai and Scan FAB patterns demonstrated above live at Layer 3 — composed patterns built from canonical tokens (touch targets, motion, typography) and semantic tokens (classification colours, state colours). The Gemini AI classification and Sarvam Bulbul TTS are service integrations — the design system provides the UI contract; the AI services provide the intelligence. This separation means departments can swap inference providers without changing the interaction pattern.

Economic case: 50+ central government portals, each spending ₹20–30 lakhs on foundational UI/UX that a shared system would obviate. Add state governments — the duplicated effort runs to crores annually. But the real value isn't cost savings. It's capability democratisation: a district-level NIC team in Mizoram ships the same quality as a ₹50 crore consultancy in Delhi.

Global South export: A design system that survives India's extremes — 14 languages, 13 scripts, ₹6,000 devices, 300M neo-literate users — is exportable to any developing nation. The method (one design authority + AI coding tools = population-scale system) becomes replicable infrastructure for Bangladesh, Nigeria, Indonesia. India leads by example, again.
